## 0. AI 操作协议(必读)

你正在阅读一份AI 接力包（research information pack）。它是研究员交付的"问题特化的决策上下文",不是开放素材库。使用时遵守以下协议,违反任一条会导致输出不可用。

### A. 引用与溯源

- P1. 任何来自本包的事实/原声/数据/分群/痛点/约束，输出时必须保留来源 ID（`Q-N` / `D-N` / `L-N` / `M-N` / `G-N` / `P-N` / `CON-N` / `S-N` / `F-N` / `C-N` / `U-N`）。**ID 含义见 frontmatter `id_schema`，不在协议正文复述。**
- P2. **无 ID 即拒绝输出**：这是 §0 C2"超出本包覆盖"的可执行版本。下游 AI 输出任何来自本包的判断时，必须能点出对应 ID；点不出 ID = 在脑补 = 拒绝输出该判断。
- A3. 引用 `Q-` / `M-` 时可截取片段,不得改写原话语义。引用 `D-` 时必须连样本基数和时间范围一起带,如 "63%(n=47, 2026-Q1)"。引用 `L-` 时保留作者/机构 + 年份。

### B. 结论与边界

- P3. 结论以 §2 结论骨架为准。不要从 §3 素材自行归纳新结论——素材是支撑，不是源头。
- P4. 触碰 §1.2 负面清单的输出一律拒绝。输出里说明 "AI 接力包负面清单不允许此判断"。
- B3. 涉及全量用户的判断,§6 反例必须并入限定条件,不能省略。
- B4. §4.3 设计约束(`CON-N`)是非协商项,方案必须满足。
- B5. 概念用 §4.4 词典的定义。不要把用户口语和团队术语混用。
- B6. 用户分群引用 §4.1。涉及分群比例时必须注明 "基于 N=X 样本,不代表总体分布"。

### C. 缺失与未知

- C1. 任务触及 §7 未解决问题(`U-N`)列出的领域时,输出里显式标注 "研究未覆盖(U-N),以下为推断"。不允许装作有依据。
- C2. 任务问到的内容超出本包覆盖(无法点出对应 ID,见 P2):

  - 在输出里写明 "本 AI 接力包未覆盖此问题,建议向研究员求证"
  - 不要从其他来源拼凑
- C3. 看到任何 `{{...}}` 占位符或 `<!-- -->` HTML 注释 = 这份包未完成生成或已损坏。停止使用,告诉用户回研究员重新生成。

### D. 时效
