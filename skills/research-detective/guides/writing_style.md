# 写作风格规范——去 AI 味

> 本文件是研究报告落笔阶段的硬约束。research-detective 写报告前先读，research-reviewer 审查时按这份打分。
> 它不是流程指南，是**写出来的字应该长什么样**——管的是句法、词法、论证习惯。

---

## 一句话

> **不复述、不堆词、不跟空气辩论。讲事实、给判断、用句号。**

---

## 5 条心法 · 每条带红灯信号

### 1. 不为了凑结构而写

对仗、整齐数字、情绪尾巴 — 不写也讲清楚?是,就删。

🚨 **看到这些就删**:

- 「不是 X,而是 Y」反复用 → 凑金句感
- 5 个章节标题长得很像 → 按模板填空
- 段尾突然金句("这是好消息""值得思考")→ 想"收得漂亮"
- 「N 大方向 / 三方向 / 三步曲 / 三位一体 / 五大发现」 → 凑数

---

### 2. 不跟空气辩论(稻草人)

写"不是 A,是 B" 之前问自己:**A 这观点上下文里真有人提过吗?** 没有,就删 A,直接讲 B。

🚨 **看到这些就删**:

- 「这不是 X 问题,是 Y 问题」(没人说过 X 是问题)
- 「问题不在 X,在 Y」(对话里从未讨论 X)
- 「这并非 X 的事,而是 Y 的事」

---

### 3. 不堆大词,落到具体场景

大词出现时问:**这词翻译成具体动作 / 具体的人 / 具体的事是什么?**

- 翻译得出来 → 写翻译,删大词
- 翻译不出来 → 整段砍掉,本来就空

🚨 **看到这些就删**:

- 「结构性 X」「X 悖论」「X 赋能」「X 重构」「X 跃迁」
- 「X 的本质」「X 的深层」「X 的核心」
- 「被理解的 X」「X 的隐形负担」(抽象偏正结构)

---

### 4. 用句号,不用破折号

破折号 "——" 是 AI 写作最强的视觉 tell。研究员句式偏好句号短句。破折号留给真正的插入语。

🚨 **看到这些就删**:

- 同段 ≥ 2 个 "——"
- 「——而」「——但」「——这就是」反转尾巴
- 「——它就是」「——这恰好是」拖腔

---

### 5. 敢给建议,但每条建议要可证伪

**研究员要有观点。** 没观点的研究 = 信息搬运,价值打折。鼓励大胆假设——"我觉得方向是 X""建议下一步做 Y""这暗示一个还未验证的可能性 Z"——这些都该写,要敢写。

但**大胆假设 = 小心求证**。每条建议 / 假设 / 方向出现时,自带三件:

1. **证据锚** — 挂在哪些观察上(S3、S7、S11 在什么场景下表现什么)
2. **边界** — 对哪类用户、什么条件成立;本研究的样本约束是什么
3. **可证伪的下一步** — 怎么验、看什么指标变化才算假设站住

三件齐 → 鼓励;缺一 → 改成"假设"措辞并补齐,或砍掉。详见红线 6。

🚨 **看到这些就改**(不是删,是补三件):

- 「应该 X」「必须 X」「需要 X」「战略上要 X」 → 这些词可用,但后面没挂 S 编号、没标边界、没说怎么验 → 补齐三件,或改成"建议探索 X(基于 S3/S7,边界是...,下一步...)"
- 「这反映了 X 的结构性缺失」 → 大词 + 战略级判断 + 零证据 → 红线 1+6 双过线,回炉
- 报告通篇都是"22 人中 X 人提到..." 没有研究员自己的判断和方向建议 → 这是另一种过线:**信息搬运,不是研究产出**

---

## 一个判定问题

写完一段读一遍,问自己:

> **如果换一个研究员 / 一个会写字的人写,会用同样的句式吗?**

不会 → 删。

---

## 写作动作的顺序:删 → 改 → 加

1. **先删** — 凑结构的对仗、稻草人、情绪尾巴、多余破折号、大词堆砌
2. **再改** — 大词翻译成具体场景,空话变事实,长句拆短句
3. **最后加** — 真正缺的:具体人物 / 具体场景 / 反例 / 厚描

> 大部分 AI 味问题靠 **删** 就能解决一半,改和加只是补漏。

## 一、红线 11 条(绝对不能做)

### 红线 1 · 不用"概念癌"词组

**禁词清单**(出现即扣分):

| 类别 | 反例 |
| --- | --- |
| "结构性 X" | 结构性消耗 / 结构性裂痕 / 结构性困境 / 结构性失衡 |
| "X 悖论" | 核心悖论 / 个性化悖论 / 健康管理悖论 / 技术赋能的核心悖论 |
| "X 重构" | 角色重构 / 关系重构 / 认知重构 |
| "X 赋能" | 技术赋能 / AI 赋能 / 数据赋能 |
| "X 跃迁" | 认知跃迁 / 价值跃迁 |
| "X 张力 / X 矛盾"作大词用 | 深层张力 / 根本矛盾(可用,但必须立刻挂回具体场景) |
| 抽象偏正结构 | 被理解的掌控感 / 情绪劳动的隐形负担 / 学习场景中的双重挫败 |
| "X 的本质 / X 的深层" | X 的本质诉求 / X 的深层心理 / X 的底层逻辑 |
| AI 抒情动词(对应 humanizer #7 AI vocabulary) | 彰显 / 铸就 / 谱写 / 织就 / 勾勒出 |

**判定标准**:这个词如果能直接出现在咨询公司 PPT 标题里,大概率是概念癌。

**为什么**:这些词对读者来说**信息量为零**——"结构性消耗"既不告诉读者"哪种消耗",也不告诉读者"为什么是结构性的"。它们是 AI 在用大词包装"我有判断"的姿态,而不是判断本身。AI 抒情动词("彰显""铸就""谱写")是同一类问题的中文版,英文里对应 underscore / showcase / tapestry——研究报告里出现 = 修辞包装,不是事实陈述。

---

### 红线 2 · 不用"不是 X,而是 Y"句式套路

**反例**:

- 「问题的核心**不是**功能缺失,**而是**流程错位」
- 「**并非**意愿不足,**而是**缺乏即时正反馈」
- 「**不是**'拥有更多',**而是**'减少消耗'」
- 「店主要的**不是**更智能,**而是**更皮实」

**禁令**:同一份报告内**最多出现 1 次**。重复出现即是模板套路。

**为什么**:这个句式制造的是"金句感",不是真发现。第一次读看起来惊艳,第五次读就明显是 AI 修辞模板。**真正的 finding 不会五条都长成同一句法**——这是"修辞模板批量生产"的最强 tell。

> 注:"店主要的不是更智能,而是更皮实"如果作为一份报告里**唯一一次**出现的金句,可以保留。

---

### 红线 3 · 章节标题不同构

**反例**(5 个二级标题完全同构):

```text
× 「X 已成为 Y,但 Z」
× 「X 的 Y 动机——既是 A,更是 B」
× 「X 中的 Z 困境——A 未能 B」
× 「X 鸿沟——Y 沦为 Z」
× 「X 中的双重挫败——既不懂 Y,也不懂 Z」
```

5 段标题完全同一种句法模板。

**禁令**:同级章节标题,**句法结构不能完全相同**。如果 5 个二级标题都是"X 的 Y——A 与 B",回炉重写。

**为什么**:真发现的标题应该长不同样子,因为现实里的发现就是不同样子。同构标题暴露的是"AI 在按一个模板填空"。

---

### 红线 4 · 不用"凑整齐"的数字

**禁词**(数字偏好):

- 三方向 / 三步曲 / 三位一体 / 三大支柱
- 五大发现 / 五重困境 / 四大趋势

**判定标准**:报告里出现的数字(三 / 五 / 四),要能回答"为什么是这个数,不是别的数"。如果数据本来支持 4 类,不要硬切成 5 类;数据支持 7 类,不要压成"三大类"。

**反例**:

- 「6 个工作类别」「6 段工作年限」——为了凑整齐,出现"1 人""1 人"这种长尾凑数
- 「执行摘要五大发现」——5 条层级混乱(结构层 / 机制层 / 归因层 / 诉求层 / 场景层)

**为什么**:整齐的数字让读者怀疑"这是为了好看硬切的"。**数量由数据决定**,该 2 类就 2 类,该 7 类就 7 类。如果非要切成整数,标题里**不要把数字暴露出来**——叫"主要发现"而不是"五大发现"。

---

### 红线 5 · N<30 不用百分比

**禁令**:N<30 时,**不允许**生成任何 X% 表达。

**反例**:「47% 的受访者认为...」「58% 的店主表示...」「65.0% 选择 A 方案」

**替代**:

- 用计数:「22 人中 9 人提到...」
- 用程度词:「多数 / 少数 / 个别 / 半数以上」
- 用案例:「在 S3、S7、S11 等案例中...」

**为什么**:百分比把小样本里 22 个具体的人压成均质团块,**违反了定性研究的根本约束**。这条与 CLAUDE.md「Research Rules §3 结论强度匹配证据强度」联动——百分比是"强证据"语感,小样本撑不起来。

---

### 红线 6 · 建议要有观点,但要可证伪

**研究员要有判断,要敢给建议——这是研究的价值所在。** 没观点的研究 = 信息搬运。这条红线不是禁建议,是禁**不挂证据、不标边界、不可证伪**的拍脑袋建议。**鼓励大胆假设,要求小心求证。**

**反例**(过线,不是因为"给了建议",而是因为建议悬空):

- 「技术应赋能'做减法'而非'做加法'」(纯口号——没说基于哪些观察、对哪类用户、什么条件成立、怎么验证)
- 「产品设计需正视这一矛盾:不仅要优化 AI 的共情能力,更要思考如何重建人际支持网络」(战略级判断,但不挂证据、不标边界)
- 「47% 的受访者认为 CRM 是录入负担,产品设计需正视这一矛盾」(N<100 直接拍"产品设计需",越过证据强度;且小样本不应给百分比——见红线 5)

**判定标准**:每条建议(尤其是含"应该 / 建议 / 方向 / 可探索 / 假设"等词时)必须三件齐全:

1. **证据锚**——挂在哪些观察上(S3/S7/S11 在 X 情境下表现 Y)
2. **边界 / 不确定性**——对哪类用户、什么场景成立;本研究的样本约束是什么(N、构成、覆盖范围)
3. **可证伪的下一步**——若做 W,先看 V 是否还成立;或建议做什么验证、看什么指标变化

三件齐全 = 鼓励;缺一 = 过线。

**正例**(大胆假设 + 小心求证):

- 「**假设**:对于 S3、S7、S11 这类高频对账店主,补'每日营业小结自动推送'可能比'优化录入流程'更能减负——他们的痛点不是录入慢,是录完看不到结果(三人原话:'录完就忘了'/'看不到回报'/'像白干')。**边界**:本研究 22 人,其中 9 人是连锁店店长;**下一步**:做一个最小版本,5 家店一周回测,看'连续 7 天录入率'是否提升 30%+——若不提升,假设可证伪。」
- 「**大胆假设**:店主不需要更多 AI 智能,需要的是'被系统记住'(昨天做了什么、今天系统接得上)。证据:S5、S11、S15 用'像没人管'/'白干'描述当前体验。**这是 22 人样本下的假设,不是结论**——建议下一步做 A/B 测:版本 A 加智能建议、版本 B 加'昨日回顾',对比 30 日留存。」

**为什么**:N<100 的定性研究是**生成假设**,不是**最终判定**——所以建议写成"可证伪的假设 + 证据锚 + 边界 + 下一步验证",而不是"应该 / 必须"的悬空口号。这条与 CLAUDE.md「Research Rules §3 结论强度匹配证据强度」「§4 分清事实/观察/解读/建议」联动:建议层是允许的、是必要的,但要标清楚它是建议层,并附上验证锚点。

**与旧规则的差别**(给后续审查者):

- 旧版禁"应该 / 必须 / 需要 / 战略"等词 → 过严,把研究员的判断也禁掉了
- 新版:这些词可用、要敢用,**但每次出现必须配证据锚 + 边界 + 可证伪锚点**

---

### 红线 7 · 章节标题必须是 finding,不是描述

**反例**(描述性目录):

- 「门店日常经营的状态与压力图景」
- 「会员系统在门店运营中的角色与痛点」

这些是**目录**,不是**发现**。读者读完只知道"这章在讲什么范围",不知道"研究员的判断是什么"。

**正例**(finding statement):

- 「会员系统从'营销工具'被店主用成了'临时记账本'」
- 「想认真做对账又坚持不下:执行失败不是意愿问题,是反馈回路问题」

**判定标准**:章节标题如果**自身就是一个判断句**,过线。如果只是名词短语堆砌,回炉。

**替代结构**:**"主标题:finding statement"** 双层

- 主标题:章节范围
- 副标题:核心发现
- 例:「会员系统使用场景:从被动响应到主动管理的转变」

---

### 红线 8 · 摘要不堆引语

**反例**:
> 「47% 的受访者认为 CRM 是无操作负担的录入工具 [20]、[2]、[6]、[7]、[16]——这反映了门店工具体系的结构性缺失。产品设计需正视这一矛盾...」

一段摘要里出现 5 个跳转角标 + 1 个产品建议——把"摘要"做成了"摘要 + 论据 + 启示"的迷你论文。

**位置约束**(不是禁建议,是分层):执行摘要的每一条 finding **不放原始引语角标**(下沉到主体章节做支撑),**不挤进产品建议**——建议要敢给(见红线 6),但放在报告末尾的"启示 / 假设与下一步"章节,在那里完整展开"证据锚 + 边界 + 可证伪下一步"。摘要里硬塞一句战略口号 = 既没空间挂证据,又抢了 finding 的位置。

**替代**:

- 摘要每条 ≤ 40 字
- 只放 finding statement
- 详细论据 / 引语 / 启示下沉到主体章节
- 摘要后可加"展开论据 ▾"二级折叠,而不是直接堆在主文里

**为什么**:摘要是 30 秒判读的钩子。读者读完 5 条摘要应该能记住 5 个判断,而不是 1000 字的迷你论文。**堆引语用"看,多个人都这么说"替代"分析推演",这是定量的支撑方式**。

---

### 红线 9 · 不否定一个"没人提过"的观点

**反例**:

- 「这份报告暴露的**不是**模型能力问题,**是**产品建模错了」(没人说过模型有问题)
- 「这**不是**写作技巧的问题,**是**思维方式的问题」(没人提过写作技巧)
- 「**问题不在**样本量,**在**分析方法」(对话里从未讨论样本量)

**禁令**:不要"先否定一个没人说过的观点,然后给出自己的观点"。

**判定方法**:写完一句"不是 A,是 B" 之后,问自己——**A 这个观点真的有人提过吗?是上下文里的真实争论焦点吗?** 如果不是,A 是 AI 自己造的稻草人,删掉。

**替代**:直接讲 B,不预设虚构的对立面。

- ❌「**这份报告暴露的不是模型能力问题,是产品建模错了**——它用做定量问卷的逻辑做深度访谈分析」
- ✅「**这份报告把定性数据当定量问卷处理了**——逐题切片归纳、22 人样本算百分比、把招募阶段的样本特征当研究主题。这是产品对'什么是定性研究报告'建模错了的结果。」

**为什么是 AI 的套路**:这是 LLM 在"制造对比张力"——通过先树立一个稻草人靶子,让自己的真实判断显得更"有洞见"。但读者会有不舒服的体感:**"等等,我从没说过这是模型能力问题啊"**——AI 在跟一个不存在的对话者辩论。

**这条与红线 2 的区别**:

- 红线 2 是**句式套路**(任何"不是 X,而是 Y" 重复出现都触发,即使 X 是真的对立观点)
- 红线 9 是**论证逻辑套路**(否定一个没人提过的观点,即使只用一次也违规)

两条**叠加触发**的情况最严重——既是稻草人否定,又是固定句式,读者一眼就能识别"这是 AI 写的"。

---

### 红线 10 · 少用破折号"——",用句号

**反例**(AI 满篇都是这种):

- 「47% 的受访者认为 CRM 是无操作负担的录入工具**——**这反映了门店工具体系的结构性缺失**——**产品设计需正视这一矛盾...」
- 「[CRM 工具 X] 没做这层**——**而这恰好是 [CRM 工具 X] 想吃下的客户」

**判定标准**:

- 同一段出现 ≥ 2 个 "——" = 触发
- 用 "——而" / "——但" / "——这就是" 开启反转感的小尾巴 = 触发(不论几次)

**禁令**:能用句号的地方,优先用句号。"——" 留给真正的插入语 / 真正的同位语,不用来做"金句感分句"或"反转感拖腔"。

**替代**:

- ❌「专业研究员一眼能识破——而 [CRM 工具 X] 主要服务的就是这批客户」
- ✅「专业研究员一眼能识破,直接判定 [CRM 工具 X] 不专业。而 [CRM 工具 X] 主要服务的就是这批客户。」

**为什么**:破折号是 AI 写作最强的视觉 tell 之一。研究员句式偏好句号短句,直接陈述,不靠 "——" 制造节奏。一段里 ≥ 2 个 "——" 几乎必然是 AI 在用同一个修辞模板加戏。

---

### 红线 11 · 不用春秋笔法(让步缓冲词 + 时间副词偷渡判断)

**春秋笔法 = 不直接下判断,而是用语气词、定语、缓冲层暗示立场**,让"细心的人"读出另一层意思,"不细心的人"被误导。AI 比人更系统地犯——RLHF 训过它"避免直接下判断 / 保持平衡 / 软化措辞",结果就是默认输出春秋笔法。

**反例 a · 让步缓冲词**(出现即触发,这些词在严肃研究报告里没有合理用法):

- 「**诚然**,AI 工具有局限性,但其潜力不可忽视」
- 「**不可否认**,X 存在问题」
- 「**客观地说**/**客观而言**,店主对工具的态度是积极的」
- 「**相对而言**,这一群体表现更稳定」
- 「**平心而论** / **毋庸讳言** / **毋庸置疑** / **毫无疑问** / **无可厚非**」

为什么是硬指纹:这些词在评论员/演讲腔里常见,在研究报告里**永远能替换得更直白**:

- 「诚然 A 有道理,但 B 更关键」→「A 有道理,B 更关键」
- 「不可否认 X 存在问题」→「X 存在问题」(有证据)/「X 是否是问题待验证」(没证据)
- 「相对而言这一群体更稳定」→「相对 [谁] 这一群体更稳定?」必须挂参考系

**反例 b · 时间副词偷渡判断**(把判断伪装成定语,藏掉举证责任):

- 「**长期被**忽视的对账难题终于得到关注」
- 「**一直被**低估的真实需求」
- 「**迟迟未**得到响应的痛点」
- 「**始终未能**落地的方案」

为什么是硬指纹:「长期/一直/迟迟/始终」是**时间副词**——研究员说"长期"就要给具体年限/数据支撑("过去 3 年财报显示..."),不能伪装成形容词扔在那。「长期被忽视」既不告诉读者"忽视了多久",也不告诉读者"谁忽视的""怎么知道是忽视而不是无关",信息量为零。

**判定**:让步缓冲词 + 时间副词定语,**出现即触发**。

**为什么**:春秋笔法的危害是产生"歧义、推诿责任、不够直白"——读者读完不知道研究员的判断是什么。研究员的价值在于**敢给判断**(见红线 6),春秋笔法是与之相反的偷懒模式:既想给立场,又不想为立场负责。

> 注:本条与黄线 7(假被动密度)联动——后者抓"被认为/被视为"密度,是春秋笔法的另一种语法形态(隐藏施动者),作为启发式黄线,允许偶尔使用,但单段 ≥3 次 = 系统性隐藏。

---

## 二、黄线 7 条(尽量避免,有理由可破例)

### 黄线 1 · 不用"用户们 / 她们 / 他们"作主语

**反例**:「**她们**普遍认为...」「**用户们**有 X 倾向...」

**问题**:把 22 个具体的人压成均质团块。

**替代**:

- 「22 人中 9 人提到...」
- 「夫妻店店主群体里(S3、S7、S11)有 X 表现」
- 「S5 是反例:在 X 情境下表现相反」

**例外**:在做明确的"全样本观察"时可以用,但必须紧跟具体计数或类型学切片。

---

### 黄线 2 · 不用公文化连接词 / signposting 假权威开场

**反例**:

- 公文化连接词:「**通过**深度访谈,**旨在**揭示...」「**借由**这一发现,可以**进一步**推导...」
- signposting / 假权威开场(对应 humanizer #28 Signposting + #27 Authority tropes):「**综上所述**...」「**归根结底**...」「**说到底**...」「**值得注意的是**...」「**不难看出**...」「**毋庸置疑**...」「**显而易见**...」「**不言而喻**...」

**替代**:

- 用动词直接句:「访谈了 22 位店主,发现...」「这意味着...」
- 直接说结论,不要用语气词预告"我接下来要做总结/给判断了"——读者已经能看到下一句

**为什么**:公文连接词是 AI 模仿"学术报告"语感的产物。signposting 类是 AI 在用"综上所述/不难看出"假装做了论证——真正的研究员不会用语气词替代论证,他们直接给证据。"值得注意的是"在英文报告里对应"It is worth noting that",humanizer 把这类称为 persuasive authority tropes("At its core...""The truth is...")——靠语气抬高说服力,不靠证据。

---

### 黄线 3 · 同义词堆叠不算论证

**反例**:「**深刻地、本质上、根本上、底层地**反映了...」

**判定**:连续 2 个同义副词出现 = 黄线触发。

**替代**:挑最准的那个,删掉其他。

---

### 黄线 4 · 章节末尾不"金句收尾"

**反例**:

- 「技术应赋能'做减法'而非'做加法'」
- 「这不仅仅是技术问题,更是一种深层的认知重塑」

**问题**:每章末尾都试图收一句"金句",造成读者疲劳。

**替代**:章节末尾用**承上启下的过渡句**(指向下一章),或**留白**(让读者自己消化)。整份报告里**金句最多 1-2 句**,出现在执行摘要或最后的启示部分。

---

### 黄线 5 · 不预设"用户都是有问题的"语气

**反例**:「**她们的困境在于...**」「**普遍存在的痛点是...**」

**问题**:AI 默认会把所有访谈结果框进"问题/痛点/困境/挑战"语义场。

**替代**:中性观察:「她们在 X 情境下表现为 Y」「这个行为同时具备两面性:A 是收获,B 是代价」

**为什么**:定性研究不是"挖痛点列表",是"理解机制"。预设"用户都有问题"会过滤掉**用户自己觉得不是问题的事**——而那些往往才是真正的洞察。

---

### 黄线 6 · 不在一句话里堆 ≥3 处加粗(对应 humanizer #15 Boldface overuse)

**反例**:

- 「**研发投入** 应该聚焦在 **核心场景**,并优先 **打磨细节**,最终 **建立壁垒**」(单行 4 处加粗,加粗失效)
- 「**用户**在 **使用流程**中遇到 **认知摩擦**,导致 **流失率上升**」(每个名词都加粗 = 没有重点)

**问题**:加粗的作用是让读者扫到"全篇里最关键的几句"。一句话堆 3 个加粗 = 没有重点 = 加粗信号失效,读者眼睛会自动跳过。这是 AI 写作的强 tell——humanizer 把它列为 #15 Boldface overuse,原例:「**OKRs**, **KPIs**, **BMC**」。

**判定**:**单行内** `**xxx**` 出现 ≥ 3 次 → 触发。注意是单行,不是单段——markdown 列表里每行一个加粗关键词是合法用法(工具文档/要点列表常见,不算滥用)。

**例外**(明确豁免):

- 元信息行:「**研究项目**: 测试 **完成日期**: 2026-06-08 **样本**: N=22」——这种"标签:值"块不算正文加粗
- 表格内 `|` 分隔的多列加粗

**替代**:全句最多保留 1-2 个最关键的加粗;其余删掉,让读者自己读出强调。如果一句话里真有 3 个并列要点,改写成列表项,每行一个。

---

### 黄线 7 · 单段假被动密度(春秋笔法的语法形态)

**反例**:

- 「研发团队**被认为**应该重新规划。设计方向**被视为**关键。优先级**被普遍认为**需要调整。」(单段 3 处假被动)

**问题**:被动语态默认隐藏施动者——「**谁**认为研发团队该重新规划?访谈里某位店主?某份内部讨论?还是研究员自己的判断?」。研究员的价值在于明确"这是谁说的、这是不是我的判断",连续被动 = 把责任偷渡给"匿名共识",这是春秋笔法的另一种语法形态(对应红线 11 是词汇形态)。

**判定**:**单段内** `被认为 / 被视为 / 被看作 / 被普遍认为 / 被广泛认为` 合计 ≥ 3 次 → 触发。

**为什么是黄线**:被动语态在描述实验过程或客观流程时有合法用法("样本被随机分配""数据被记录"),≥ 3 次密度阈值能区分"偶尔用"和"系统性隐藏"。允许人工破例(说明被动确实更准确),但要在该段写明施动者。

**替代**:

- ❌「研发团队**被认为**应该重新规划」
- ✅「访谈中 4 位店主(S3/S7/S11/S15)明确表示研发应聚焦门店端」
- ✅「我判断研发应聚焦门店端,基于 [证据 1]+[证据 2]」

## 四、给报告生成 prompt 的硬约束

> 当你作为 LLM 在生成研究报告时,把以下条款作为内置约束。

```text
=== 写作约束 ===

[禁令]

1. 禁用词组:"结构性 X"、"X 悖论"、"X 赋能"、"X 重构"、"X 跃迁"、"X 的本质"、"X 的深层"、"被理解的掌控感"等抽象偏正结构
2. 禁用句式:"不是 X,而是 Y"——同一份报告内最多出现 1 次
3. 禁用"五大发现 / 三大方向 / 三步曲 / 三位一体"等数字标题
4. 禁用"她们 / 用户们 / 多数受访者"作主语,改用具体计数(N 中 X 人)
5. 当 N<30,禁止生成任何百分比表达
6. 章节标题禁止 5 个同级标题用同一句法模板
7. 章节标题禁止纯描述性(必须是 finding statement)
8. 执行摘要每条限 ≤ 40 字,不堆引语角标,不在摘要里塞战略口号(建议下沉到末尾"启示 / 假设与下一步"章节完整展开)
9. 章节末尾不强制金句收尾
10. "应该 / 建议 / 方向 / 假设"等判断词可用、要敢用,但每次出现必须三件齐:① 证据锚(挂回 S 编号 + 场景)② 边界(对哪类用户、什么条件成立、样本约束)③ 可证伪的下一步(怎么验证、看什么变化才算假设站住)。三件缺一 = 越界口号
11. 禁用稻草人句式:不要先否定一个对话上下文里没人提过的观点,再讲自己的判断
12. 少用破折号"——":能用句号的地方用句号。同一段最多 1 个 "——";禁用 "——而" / "——但" / "——这就是" 这种反转尾巴

[要求]

1. 每个判断必须挂回到具体受访者编号(S3、S7 等)和具体场景
2. 至少识别并标注 1 个反例 / 异常值 / 张力(用"但 S5 是反例:...")
3. 用动词直接句,不用"通过 X,旨在 Y"公文化连接词
4. 引语必须 ≥ 1 句完整,不要单字"够用""不行"
5. 数字数量由数据决定,不预设"X 大方向"
6. 核心结论旁标注证据强度标签:[N人提及 / M反例 / 跨K类来源],让非研究员读者秒判可信度

[正例参考]
✓ "22 人中 9 人提到 X,其中 S3、S5、S11 给出了具体场景:[一段厚描]。但 S7 是反例,在 X 情境下表现相反。"
✓ "店主想认真做对账又坚持不下:执行失败不是意愿问题,是反馈回路问题"

[反例参考]
✗ "47% 的受访者认为 CRM 是无操作负担的录入工具——产品设计需正视这一矛盾"
✗ "技术赋能的核心悖论与突破方向"
✗ "不是更智能,而是更懂事 [此句法已用过 4 次]"
```

### 词法层

- [ ] "不是 X,而是 Y"全文最多出现 1 次
- [ ] 同级章节标题句法**不同构**
- [ ] 没有"用户们 / 她们 / 多数受访者"做主语(或紧跟具体计数)

### 论据层

- [ ] N<30 时**没有任何百分比**
- [ ] 每个判断挂回到具体 P 编号 + 具体场景
- [ ] 至少有 1 个反例 / 异常值标注
- [ ] 核心结论旁标注证据强度标签 `[N人提及 / M反例 / 跨K类来源]`

### 结构层

- [ ] 章节标题是 finding statement,不是描述性目录
- [ ] 执行摘要每条 ≤ 40 字,不堆引语,不挤进战略口号(建议下沉到末尾"启示"章节展开)

### 建议层(鼓励有观点,但要可证伪)

- [ ] 报告里有研究员自己的判断和建议,不是纯信息搬运
- [ ] 每条建议 / 假设 / 方向 都齐三件:**证据锚**(S 编号 + 场景) + **边界**(适用条件 + 样本约束) + **可证伪的下一步**(怎么验、看什么指标)

### 摘要级

**坏**:
> "47% 的受访者认为 CRM 是无操作负担的录入工具 [20]、[2]、[6]、[7]、[16]——这反映了门店管理工具的结构性缺失。产品设计需正视这一矛盾:不仅要优化录入流程,更要思考如何重建门店日常工作流。"

**坏**:「会员系统在门店日常中的角色与痛点」(目录)

**好**:「会员系统从'营销工具'被店主用成了'临时记账本'——他们既离不开也不想认真用」(发现)

### 章节内容级

**坏**:
> "**门店管理的隐形负担**——店主在日常经营中承担了大量未被工具支持的杂事。这种**结构性消耗**在夫妻店中尤其严重,**不仅是**人手分配问题,**更是**经营角色的深层矛盾。"

**问题**:概念癌堆叠 + 抽象套抽象 + 没有任何具体的人

**好**:
> "S11 是开了 6 年的社区咖啡店店主。我们去访谈那天是周一早上,他一边收外卖订单、一边补冰柜库存、一边给老顾客打招呼,中途 POS 死机了一次,他蹲在收银台底下重启,前后被打断 3 次。他自己说'就是没有一段时间能完整地做一件事'。
>
> 这个状态不是 S11 一个人的——S3、S7、S14 在不同时段都描述了类似的'打断感'。但 S5 是反例:她的店是预约制下午茶,她明确说'客人来之前我有 2 小时是连贯的',原因是她的客流节奏可控。
>
> 这提示我们,店主'觉得累'的根因不一定是'活多',而可能是'被打断次数'——打断次数才是消耗注意力的关键变量。"

## 七、机器可查 / 人工自检 对照总表

研究报告交付前两道关：[scripts/lint_report.py](../scripts/lint_report.py) 跑机器可查的部分（17 条规则），剩下的人工对照本文件 + [report_principles.md](report_principles.md) 自查清单。

### 命名约定

- `R_*` = **红线**：黑名单 / 正则零误判，触线即拒交付
- `Y_*` = **黄线**：启发式或可保留，人工复核给出"为什么不是套路"的理由

### 17 条规则按 5 层组织

**词法层**（黑名单可查）：

| writing_style.md | rule_id | 检查内容 |
| --- | --- | --- |
| 红线 1 · 概念癌词组 | R_VOCAB | 结构性 / 赋能 / 跃迁 / 彰显 / 铸就 等禁词 |
| 红线 4 · 凑数标题 | R_FILLER_NUM | "五大发现 / 三方向 / 三大支柱"等凑数词 |
| 黄线 2（拆出）· 公文连接词 | R_BUREAU | 通过 X 旨在 Y / 借由 / 进而推导 / 从而揭示 |

**句法层**（句式套路）：

| writing_style.md | rule_id | 检查内容 |
| --- | --- | --- |
| 红线 2 · 不是 X 而是 Y | R_PARALLEL | 句式套路全文 >1 次 |
| 红线 1（拆出）· 抽象偏正 | R_ABSTRACT | 被 X 的 Y / X 的隐形 Y / X 的双重 Y |
| 红线 10（拆出）· 破折号反转拖腔 | R_DASH_TAIL | ——而 / ——但 / ——这就是 等确定性短语 |

**春秋笔法层**（暗示立场，不直接下判断）：

| writing_style.md | rule_id | 检查内容 |
| --- | --- | --- |
| 红线 11（拆出）· 让步缓冲词 | R_HEDGE_BUFFER | 诚然 / 不可否认 / 客观地说 / 相对而言 等 |
| 红线 11（拆出）· 时间副词偷渡定语 | R_HEDGE_TIME | 长期被 X / 一直被 X / 迟迟未 X / 始终未能 X |
| 黄线 2（拆出）· signposting | R_HEDGE_SIGNPOST | 综上所述 / 值得注意的是 / 不难看出 / 显而易见 等 |
| 黄线 7 · 假被动密度（启发式） | Y_HEDGE_PASSIVE | 单段 ≥3 处"被认为/被视为/被看作"（阈值待校准） |

**定量纪律层**（研究报告独有，humanizer 不覆盖）：

| writing_style.md | rule_id | 检查内容 |
| --- | --- | --- |
| 红线 5 · N<30 不用百分比 | R_SMALL_N | 自动识别样本 N，N<30 时所有 % 报警 |
| 红线 8 · 摘要不堆角标 | R_FOOTNOTE | 摘要章节单段 ≥5 个 \[n\] 角标 |

**结构 / 视觉层**（启发式黄线，可能误判，人工复核）：

| writing_style.md | rule_id | 检查内容 |
| --- | --- | --- |
| 红线 10（拆出）· 段内多破折号 | Y_DASH_DENSITY | 段内 ≥2 个 ——（启发式） |
| 黄线 6 · 单行加粗滥用 | Y_BOLD | 单行 ≥3 处 \*\*xxx\*\*（启发式） |
| 红线 3 · 章节标题不同构 | Y_TITLE | 同级标题骨架对比（启发式，骨架抓不全语义同构） |
| 黄线 1 · 抽象主语 | Y_GENERIC | 她们/用户们 + 概括副词作主语 |
| 黄线 4 · 章节末尾金句收尾 | Y_CLOSING | 章末规范性祈使 + 关闭性短语 |

### lint 不覆盖的（必须人工对照）

| writing_style.md | 为什么 lint 抓不到 |
| --- | --- |
| 红线 6 · 建议要可证伪 | 论证层——证据锚 + 边界 + 下一步验证需要语义判断 |
| 红线 7 · 标题必须是 finding | 论证层——"标题是判断还是描述"需要语义判断 |
| 红线 9 · 不否定没人提过的观点 | 论证层——稻草人识别需要回到原始证据 |
| 黄线 3 · 同义词堆叠 | 同义词识别需要语义判断 |
| 黄线 5 · 不预设"用户都有问题"语气 | 语气倾向需要语义判断 |
| 红线 11（无法机器化的部分） | 详略不对称 / 语序暗示因果 / 选择性引用——这些是论证层春秋笔法，需要 reviewer + 人对抗审查 |

### lint 已知豁免

不该用 lint 卡住，也不要为了讨好 lint 改文件：

- 引号内（「」『』""''《》）和 blockquote（`>`）里的春秋笔法词 → 视为引用原话或示例，自动豁免
- writing_style.md 本身（禁词词典需要列举反例）
- simple_report.md 模板（占位符 `——建议方法:` 是语义结构）

### lint 命令

```bash
python3 skills/research-detective/scripts/lint_report.py outputs/<报告文件>.md
```

退出码 0 = 干净通过，1 = 命中红线/黄线，2 = 文件错误。红线非零必须改完才交付；黄线人工复核给出非套路理由可保留。

## 八、本规范在工作流里的位置

| 阶段 | 怎么用 |
| --- | --- |
| **research-detective 写报告前** | 先读本文件的"心法 5 条 + 红线 11 条 + 黄线 7 条",作为生成约束 |
| **research-detective 写报告后** | 跑 lint（17 条机器可查规则）+ 对照本文件第七节人工自检清单（5 条 lint 抓不到的论证层规则），任一不过线回炉 |
| **research-reviewer 审查时** | 在 review.md 的「交付检查」里跑 lint 并粘贴输出 + 对照第七节人工自检清单 |
| **训练数据层** | 「好坏对比」章可作为对比学习样本 |

### 这份规范不是终点

它是**症状治疗**——堵住 AI 当前最明显的几个套路。但**只要 AI 没拿到具体上下文(项目背景 / 受访者状态 / 研究员意图),它就会继续以新的修辞模板复发**。

长期解法是产品建模层:让研究员在生成前补充研究目标拆解 + 案面信息,在生成后做 inline 标注反馈。本规范是这两步落地之前的过渡方案。
